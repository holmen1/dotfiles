# XKB

X11 keyboard customisation using **XKB** and **xcape**.

---

## Key behaviour

| Key      | Tap     | Hold        |
|----------|---------|-------------|
| CapsLock | Escape  | Control     |
| Super    | ‚Äî       | Num-layer   |

### Num-layer (Super held)

```
  u  i  o  ‚Üí  7  8  9
  j  k  l  ‚Üí  4  5  6
  m  ,  .  ‚Üí  1  2  3
     n     ‚Üí  0
```

## Files

| Path                                      | Purpose                                |
|-------------------------------------------|----------------------------------------|
| `dotfiles/xkb/.config/xkb/keymap.xkb`    | Compiled keymap (generated by build)   |
| `dotfiles/xkb/apply-keys.sh`             | Loads keymap + starts xcape            |
| `install/build/xkb/symbols/local`        | XKB partial symbols ‚Äî num-layer digits |
| `install/build/xkb/build-xkb.sh`         | Builds keymap.xkb from symbols/local   |

---

## Installation (Arch Linux)

```sh
sudo pacman -S xorg-setxkbmap xorg-xkbcomp xcape
```

Symlink `dotfiles/xkb/` files to `$HOME`, then build the keymap once:

```sh
# Once (or whenever symbols/local changes)
install/build/xkb/build-xkb.sh

# On every X startup ‚Äî called automatically by ~/.xinitrc
~/.local/bin/apply-keys.sh
```

## How it works

### CapsLock ‚Üí Control / Escape

`ctrl:nocaps` (built-in XKB option) replaces CapsLock with `Control_L`.  
xcape then synthesises `Escape` when `Control_L` is released without a chord:

```sh
xcape -e 'Control_L=Escape'
```

### Super ‚Üí Num-layer

`lv3:lwin_switch` (built-in XKB option) makes the left Super key the
**level-3 activator** ‚Äî the same mechanism as AltGr.  
`~/.config/xkb/symbols/local` assigns digits at level 3 for the numpad cluster.

### Build (`install/build/xkb/build-xkb.sh`)

Run once, or after changing `install/build/xkb/symbols/local`:

1. `setxkbmap -print` emits the base keymap description to stdout
2. `sed` appends `+local(numpad)` to the `xkb_symbols` include line
3. `xkbcomp` compiles the result into `dotfiles/xkb/.config/xkb/keymap.xkb`

### Load (`apply-keys.sh`)

Run on every X startup via `.xinitrc`:

1. `xkbcomp keymap.xkb $DISPLAY` loads the pre-compiled keymap ‚Äî no setxkbmap, no sed
2. `xcape` is restarted to synthesise Escape on CapsLock tap

## Debugging

```sh
setxkbmap -query                                   # current options
xkbcomp -xkb "$DISPLAY" - | grep -A4 'key <LWIN>' # check Super mapping
xkbcomp -xkb "$DISPLAY" - | grep -A4 'key <AD07>' # check u/7 mapping
xev | grep -A2 KeyPress                            # watch raw events
```
